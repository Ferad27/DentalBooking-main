# DentalBooking

DentalBooking е уеб приложение за онлайн записване на часове в зъболекарска клиника. Проектът включва публична част за пациенти, регистрация и вход, записване на часове, профили на зъболекари, както и отделни табла за пациенти, зъболекари и администратори.

## Описание

Приложението предоставя модерен интерфейс за зъболекарска практика, чрез който пациентите могат да разглеждат услуги и зъболекари, да избират дата и час за посещение и да управляват своите записвания. Зъболекарите разполагат с панел за график, пациенти и история на посещенията, а администраторите могат да управляват потребители, пациенти, зъболекари, часове и справки.

Проектът е създаден с React, TypeScript, Vite, Tailwind CSS, shadcn/ui и Supabase.

## Основни функционалности

### Публична част

- Начална страница с представяне на клиниката.
- Списък с услуги и цени.
- Списък със зъболекари и филтриране по специалност.
- Детайлна страница за зъболекар.
- Форма за запазване на час.
- Регистрация и вход в системата.
- Responsive навигация за desktop и mobile.

### Пациентски панел

- Преглед на предстоящи часове.
- История на посещенията.
- Отказване на час.
- Достъп до профилна информация.
- Препоръчани услуги.
- Известия за предстоящи часове.

### Зъболекарски панел

- Преглед на предстоящи часове.
- Управление на график и работно време.
- Управление на пациенти.
- История на записванията.
- Генериране на справки за период.

### Административен панел

- Управление на потребители.
- Управление на зъболекари.
- Управление на пациенти.
- Управление на записани часове.
- Отчети и аналитика.

## Потребителски роли

| Роля | Описание |
|---|---|
| `patient` | Пациент, който може да запазва и управлява свои часове. |
| `dentist` | Зъболекар, който вижда графика си, пациентите си и историята на посещенията. |
| `admin` | Администратор с достъп до административния панел. |

Регистрационната форма позволява създаване на акаунт като `patient` или `dentist`. Администраторска роля трябва да бъде зададена през базата данни или чрез административна логика.

## Инсталация и стартиране

### 1. Клониране на проекта

```bash
git clone <repository-url>
cd DentalBooking-main
```

### 2. Инсталиране на зависимости

```bash
npm install
```

### 3. Създаване на `.env` файл

В основната директория създайте файл `.env` със следните променливи:

```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

> Не публикувайте реални ключове в публично repository. За production среда задайте променливите през настройките на hosting платформата.

### 4. Стартиране в development режим

```bash
npm run dev
```

По подразбиране Vite стартира приложението на:

```text
http://localhost:3000
```

### 5. Production build

```bash
npm run build
```

### 6. Преглед на production build

```bash
npm run preview
```

## Конфигурация на Supabase

Проектът използва Supabase за:

- регистрация и вход;
- сесии и authentication state;
- таблици за профили, зъболекари, услуги, часове и работно време;
- RLS политики;
- SQL функции.

В проекта има Supabase миграции в директорията:

```text
supabase/migrations/
```

Ако използвате локален Supabase проект, можете да приложите миграциите чрез Supabase CLI:

```bash
npx supabase start
npx supabase db reset
```

Ако използвате remote Supabase проект, изпълнете SQL файловете от `supabase/migrations/` през Supabase SQL Editor или чрез CLI workflow.

## Бележки за разработка

- Проектът съдържа както `package-lock.json`, така и `bun.lockb`. За по-стабилна работа изберете един package manager. При текущата конфигурация е най-лесно да използвате `npm`.
- Част от данните са mock/demo данни, особено в някои административни компоненти.
- `src/lib/supabase.ts` използва `.env` променливи, докато `src/integrations/supabase/client.ts` съдържа hardcoded Supabase стойности. Препоръчително е цялата Supabase конфигурация да мине през environment variables.
- Някои services използват `dentist_availabilities`, докато миграциите създават `working_hours`. При настройване на нова база проверете дали схемата съответства на кода.
- SQL функцията `create_appointment` работи с параметър `time`, докато таблицата `appointments` използва `start_time` и `end_time`. Ако ще използвате функцията активно, синхронизирайте я със структурата на таблицата.
- За admin достъп трябва потребителят да има `role = 'admin'` в Supabase профила.
